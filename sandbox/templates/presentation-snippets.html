Snippets - a really small Django CMS

In fact not even a CMS - more of a Content Management Tool

Rick Hurst

Django developer at Potato

Overview

  • an "inside out" Content Mangement Tool for Django based websites
  • Currently known as "Snippets", but too many other apps with the same name!
  • add editable areas to a django template, shared or unique
  • Why? - many full blown CMS are overkill and want to "own" your site
  • use as a CMS for small, mostly static sites - similar use case to Perch (popular, small PHP CMS)
  • drop in to any other Django CMS to provide additional editing of micro-content (e.g. straplines, footer text)

Overview pt 2

  • template tag driven - add the template tag with a unique identifier, and if the snippet doesn't exist, it is automatically created
  • allows HTML editing, as some formatting is inevitably needed
  • plan to integrate WYSIWYG HTML editor with filtering
  • "Front End Editing" via javascript pop-up (jquery plugin provided)

Future Plans

  • create djappengine version to use app engine models
  • "structured snippets" e.g. text + image with choice of micro-templates (borrow features from Perch)
  • some method of filesystem synchronisation to allow dynamic content to be put under version control
  • to keep this very small and not try to become a full CMS! Focus on making it a pluggable addition to other projects.
  • add caching (advice appreciated), as many tags = many db queries per page load.

Installation

git clone https://github.com/rickhurst/django-snippets.git

Include the following lines in INSTALLED_APPS:-

'snippets',
'snippets.templatetags',

syncdb stuff to have the SQL created for the snippet model

Use

To add a snippet to a template, first include snippet_extras:-

load snippet_extras

Then add a tag in the following format:- 

get_snippet "footer"
